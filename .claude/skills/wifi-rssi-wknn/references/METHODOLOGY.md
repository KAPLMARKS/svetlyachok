# Методология RSSI Fingerprinting

## Что такое Wi-Fi Fingerprinting

Fingerprinting (метод сопоставления радиоотпечатков) — техника indoor-позиционирования, в которой база эталонных RSSI-измерений из известных точек («радиокарта») используется для классификации позиции по новому измерению.

Алгоритм в общем виде:

1. **Калибровка (offline phase):** в каждой целевой зоне (или на сетке точек внутри зоны) выполняется N измерений RSSI от всех видимых Wi-Fi APs. Каждое измерение — пара `(zone_label, rssi_vector)`. Множество таких пар образует **радиокарту**.
2. **Локализация (online phase):** в неизвестной точке снимается одно или несколько измерений. Каждое сравнивается с радиокартой по выбранной метрике расстояния, и предсказывается класс зоны.

## Почему RSSI, а не RTT/FTM/BLE/Magnetic

В дипломном проекте принципиальный выбор RSSI зафиксирован в DESCRIPTION.md. Краткое обоснование:

| Метод | Точность | Стоимость | Применимость в проекте |
|-------|----------|-----------|------------------------|
| **RSSI Fingerprinting** | 1–5 м | Программная (без аппаратуры) | ✅ Выбран |
| RTT (IEEE 802.11mc) | 0.5–1 м | Требует совместимых APs и устройств | ❌ Старая Wi-Fi инфраструктура вуза |
| BLE Beacons (iBeacon) | 1–2 м | Закупка маяков | ❌ Не вариант для теста в вузе |
| Magnetic Fingerprinting | 1–3 м | Программная | ❌ Не Wi-Fi (тема диссертации) |
| GPS/ГЛОНАСС | 5+ м | Бесплатно | ❌ Сигнал не проходит в помещения |

## Источники шума и почему нужен ML

RSSI — нестабильная метрика. Источники флуктуаций:

1. **Multipath:** сигнал отражается от стен, мебели, людей. Конструктивная/деструктивная интерференция меняет принимаемую мощность.
2. **Body shadowing:** человеческое тело поглощает 2.4 ГГц сигнал на 4–10 dB. Положение пользователя относительно AP критично.
3. **Орientation effect:** антенна смартфона имеет неравномерную диаграмму направленности.
4. **Time-of-day variation:** в офисе с большим количеством людей RSSI «дрейфует» из-за изменения числа поглощающих объектов.
5. **AP load:** при высокой нагрузке точка доступа может снижать мощность передачи (TPC, IEEE 802.11h).
6. **Hardware variability:** разные смартфоны репортят RSSI с разной калибровкой (даже на одной модели).

Поэтому нельзя вычислить позицию аналитически (как через мультилатерацию). ML-классификаторы (WKNN, Random Forest) фильтруют эти шумы за счёт усреднения по соседним калибровочным точкам.

## Размер калибровочного сета

Для каждой зоны (workplace, corridor, meeting_room, outside_office) рекомендуется:

- **Минимум:** 30 измерений на зону (на разных смартфонах, в разное время, в разных позициях внутри зоны)
- **Хорошо:** 100+ измерений на зону, распределённые по времени дня
- **Идеально:** периодическое перекалибрование (раз в неделю-месяц) для адаптации к изменениям офиса

Если калибровочные точки распределены неравномерно по зонам, использовать `class_weight="balanced"` в Random Forest или взвешенную версию KNN.

## Векторное представление радиоотпечатка

В каждом измерении видно только подмножество всех APs здания. Подходов два:

### Подход A: Только пересечение

Использовать только те BSSID, которые видны и в test-векторе, и в калибровочном:

```python
def euclidean_intersection(a: dict, b: dict) -> float:
    common = set(a.keys()) & set(b.keys())
    if not common:
        return float("inf")  # нечего сравнивать
    return sqrt(sum((a[k] - b[k]) ** 2 for k in common))
```

**Плюсы:** интуитивно, не нужен глобальный список BSSID.
**Минусы:** не работает с sklearn KNeighborsClassifier (требует фиксированную размерность).

### Подход B: Глобальный плотный вектор (рекомендуется)

Зафиксировать множество всех BSSID, видимых в калибровочном сете. Преобразовать каждый отпечаток в вектор длины `N_bssids` с `MISSING_RSSI = -100` для отсутствующих:

```python
def to_dense_vector(observation: dict, all_bssids: list[str]) -> np.ndarray:
    return np.array([
        observation.get(bssid, MISSING_RSSI)
        for bssid in all_bssids
    ])
```

**Плюсы:** работает с sklearn, легко масштабируется, регулярная матрица для матричных операций.
**Минусы:** размер вектора = число всех видимых APs (может быть 100–500).

В нашем проекте используется **подход B**.

## Метрики расстояния для сравнения векторов

### Euclidean (L2) — по умолчанию

```python
d(a, b) = sqrt(sum((a_i - b_i)^2))
```

Стандарт для KNN. Линейная по разнице RSSI.

### Manhattan (L1, Cityblock)

```python
d(a, b) = sum(|a_i - b_i|)
```

Менее чувствительна к выбросам (одно очень разное значение не «ломает» дистанцию). Часто работает лучше Euclidean на RSSI.

### Cosine

```python
d(a, b) = 1 - (a · b) / (|a| * |b|)
```

Игнорирует абсолютную мощность, смотрит на относительные пропорции. Полезна, если RSSI калибровка между устройствами сильно различается.

В проекте: сравнить все три в экспериментах, отчитать в дипломе.

## WKNN: математика

Классический KNN: для test-вектора `x` находим `k` ближайших калибровочных точек и присваиваем класс большинства.

WKNN отличается тем, что соседи взвешиваются обратно пропорционально расстоянию:

```
weight_i = 1 / d(x, neighbor_i)
prediction = argmax_class(sum(weight_i for neighbor_i in class))
```

В sklearn: `KNeighborsClassifier(weights="distance")`.

**Эффект:** ближние соседи влияют сильнее. Это даёт более стабильный результат на границах зон.

**Подводный камень:** если test-вектор точно совпадает с одним из калибровочных (`d = 0`), `weight = inf` — sklearn это обрабатывает корректно, возвращая метку этого соседа.

## Random Forest: математика и настройки

Random Forest обучает ансамбль решающих деревьев на bootstrap-сэмплах с случайным подмножеством признаков на каждом split. Классификация — голосование большинства деревьев.

**Ключевые гиперпараметры:**

- `n_estimators=100..500` — больше деревьев, выше точность, выше время обучения. Для нашей задачи 100 обычно достаточно.
- `max_depth=None или 5..20` — `None` означает рост до листьев с минимум одним samples; для шумных RSSI ограничение глубины предотвращает переобучение.
- `min_samples_split=2..10` — минимум samples для разделения узла.
- `class_weight="balanced"` — обязательно если зоны не сбалансированы.
- `n_jobs=-1` — параллельная тренировка (использовать все ядра).
- `random_state=42` — фиксировать для воспроизводимости.

**Преимущество перед WKNN:** робастность к шуму, не требует подбора `k`, может ранжировать важность признаков (`feature_importances_`).

**Недостаток:** «чёрный ящик» — труднее интерпретировать, чем KNN.

## Сравнение WKNN vs Random Forest в дипломной

В диссертации обязательно представить:

1. **Detection Probability per zone** для каждого классификатора на одном test-сете
2. **Confusion matrix** для каждого
3. **Среднее время инференса** на одном измерении
4. **Поведение при различном размере калибровочного сета** (10%, 50%, 100% — кривая обучения)
5. **Чувствительность к выбору метрики** (для WKNN: euclidean vs manhattan vs cosine)
6. **Чувствительность к гиперпараметрам** (для WKNN: `k=1,3,5,7,9`; для RF: `n_estimators=50,100,200`)

## Дополнительные техники (опциональные расширения для диплома)

- **Усреднение N измерений за окно:** перед классификацией собирать 3–5 измерений за 5–10 секунд и усреднять RSSI по каждой BSSID. Снижает шум.
- **Калман-фильтр** для сглаживания временных серий позиций
- **Pre-filtering:** игнорировать BSSID с RSSI < -85 dBm (слишком слабые, часто шумные)
- **Hybrid WKNN+RF:** ensemble через voting (взвешенное голосование двух классификаторов)
- **Online learning:** инкрементальное обновление радиокарты по уверенно классифицированным запросам пользователей

Эти расширения можно отложить и добавить, если останется время после основной части.
