# Метрология ISO/IEC 18305:2016

> ISO/IEC 18305:2016 «Information technology — Real time locating systems — Test and evaluation of localization and tracking systems» — международный стандарт оценки точности indoor-позиционирования. В дипломе на него ссылаемся как на основу метрологического контроля.

## Ключевые метрики

### 1. Detection Probability (вероятность обнаружения зоны)

Доля случаев, когда система правильно определила зону пользователя.

```
DetectionProbability(zone) = TP_zone / (TP_zone + FN_zone)
```

где `TP_zone` — true positives (правильно отнесли к зоне), `FN_zone` — false negatives (зона на самом деле, но классификатор отнёс к другой).

**Целевые значения:**
- Workplace (рабочее место): ≥ 90% — критическая зона учёта рабочего времени
- Meeting room (переговорная): ≥ 85%
- Corridor (коридор): ≥ 75%
- Outside office (вне офиса): ≥ 95% — важно для корректного фиксирования ухода

**Формат отчёта в дипломе:** таблица с DetectionProbability по каждой зоне для каждого классификатора.

### 2. Confusion Matrix (матрица ошибок)

Квадратная матрица `N_zones × N_zones`. Элемент `M[i][j]` — число случаев, когда истинная зона `i`, а предсказанная `j`. Диагональ — правильные классификации.

**Формат отчёта в дипломе:** heatmap (через matplotlib/seaborn) с подписями зон. Показывает, какие зоны путаются между собой.

**Пример анализа:** «Workplace и corridor путаются в 12% случаев из-за общих BSSID соседнего AP №3» — основа для оптимизации калибровки.

### 3. RMSE (Root Mean Square Error)

Применима, если работаем с непрерывными координатами `(x, y)`, а не зональной классификацией.

```
RMSE = sqrt(mean((x_pred - x_true)^2 + (y_pred - y_true)^2))
```

В метрах.

В нашем проекте основная задача — зональная классификация, поэтому RMSE опциональна (если будем дополнительно строить координатную модель).

### 4. Computational Performance

- **Training time:** время полного обучения классификатора на калибровочном сете (секунды)
- **Inference time:** среднее время классификации одного запроса (миллисекунды)
- **Inference time p95, p99:** перцентили времени инференса (для оценки worst-case на проде)

**Целевые значения:**
- Inference p95 < 200 ms (мгновенный ответ для пользователя)
- Training time < 60 s на полном сете (для возможности перекалибровки в фоне)

## Тестовые сценарии «чёрного ящика»

ISO/IEC 18305 рекомендует ряд black-box тестов (воспроизводят реальные условия эксплуатации):

### Сценарий A: Стационарное измерение в каждой зоне

В каждой зоне записывается N (≥ 30) измерений в стационарных условиях (пользователь стоит/сидит на месте). Классификатор должен корректно определить зону.

**Цель:** базовая точность.

### Сценарий B: Переходы между зонами

Пользователь движется по предопределённому маршруту, пересекая зоны. Записываются непрерывные измерения (раз в 5–10 секунд).

**Цель:** оценить behavior на границах зон. Допустимо одна-две ошибки на переходах (классификатор реагирует с лагом).

### Сценарий C: Стресс-тест

Запись в условиях:
- Высокая загрузка офиса (много людей, активный Wi-Fi трафик)
- Низкая загрузка (вечер, выходные)
- Различные позы пользователя (сидя, стоя, телефон в руке vs в кармане)
- Различные модели смартфонов

**Цель:** оценить устойчивость к реальным условиям.

### Сценарий D: Долгосрочная стабильность

Записи в течение нескольких недель без перекалибровки. Точность обычно деградирует (изменения мебели, новые APs).

**Цель:** оценить, как часто нужно перекалибровать радиокарту.

## Защита от data leakage

Критично для научной работы — НЕ допустить leakage между калибровочным и тест-сетами:

- ❌ **Не делать random split одного сбора.** Калибровочный и тестовый сет должны быть собраны в **разные сессии** (разные дни, разное оборудование если возможно).
- ❌ **Не использовать GridSearchCV на test-сете.** Тюнинг гиперпараметров — только на калибровочном (через GroupKFold или временной split).
- ❌ **Не «заглядывать» в test-сет** для нормализации признаков — статистики (mean, std) считать только по калибровочному.
- ✅ **Test-сет используется только для финальной оценки.** Один раз на финальной модели.

## Сравнение классификаторов: статистическая значимость

Если разница в Detection Probability между WKNN и Random Forest составляет 1–2%, это может быть в пределах шума. Для обоснования в дипломе:

1. **Bootstrap confidence interval:** оцените 95% CI для каждого классификатора (1000 resamples тест-сета).
2. **McNemar's test** для попарного сравнения classifier accuracy на одном test-сете (`scipy.stats.contingency.mcnemar`).
3. **Cross-validation на калибровочном сете** для робастности оценки.

Если CI пересекаются или McNemar p-value > 0.05 — разница статистически незначима, и в дипломе нужно указать, что оба классификатора эквивалентны по точности.
